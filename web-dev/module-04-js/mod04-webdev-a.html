Datatyping in JavaScript is dynamic

  • If you initiate a variable eg var free; without a value JS will type it Undefined
  • When storing mixed datatypes in a variable, if you start with a string everything that comes after will be treated as a string ie var a = 10 + 15 + 'cat'; produces whereas var b = 'cat' + 10 + 15; displays
  • var c = 123e5 produces and var d = 123e-5 displays
  • Using the typof operater returns primative data types such as string, number, boolean and undefined and complex data types: function and objects (including array and null objects). Using document.getElementById("dem0").innerHTML = typeof false will return boolean
  • JS array objects are written in [] and items are accessed using [itemNumber], starting at 0
  • JS objects are declared using {} and properties are stored as name:value pairs. to access a property value, use dot notation objectname.propertyname
