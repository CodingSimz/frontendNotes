Use <meta name="viewport" content="width=device-width, initial-scale=1.0"> to ensure the initial zoom matches the device viewport (visible area). This is important for Responsive Web Design, as is making the css width property 100% for images. Images could equally benefit from max-width:100%; height:auto; so images can scale down but not up.

Forms

Client-side validation speeds things up - a better performing form than requiring server-side validation. This will speed up processing. There are a few techniques with html5 we can use for this see client-side form validation

  • Using the fieldset element is key for assistive tech, for instance next a set of radio btns inside a fieldset element. Ideally its great for sectioning forms. To label a fieldset use the legend tag.
  • Give labels the 'for' attribute eg <label for="fname">; it provides a way to bind a label to an input where the value of the for matches the input id. It also makes the a label clickable area that defines the focus of the input
  • When creating a select drop-down, use the optgroupe element to group option tags (NB) optgroup label must be used.
  • Use the required attribute to force input
  • Use the email input type to test for @
  • Use the tel input type to test for numbers and pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}" is an example of pattern matching
  • pattern="http(s?)(:\/\/)((www.)?)([a-zA-z0-9\-_]+).*$" example url pattern matching where (?) wraps optional content segments
  • Autocomplete attr works with <form> and the following input types: search, url, email, password, datepickers, range, and color
